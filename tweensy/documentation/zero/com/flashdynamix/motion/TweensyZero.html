Classpublic class TweensyZero

TweensyZero is a lightweight tweening engine designed specifically for developers who want to achieve small files sizes but are not too fused about overall performance. Though TweensyZero is still quite effecient the original Tweensy class delivers the best tweening performance.

The TweensyZero Class contains a collection of tweens specified by property tweens for an Object instance. The TweensyZero code library weighs about 2.9KB once compiled


A tween can be executed in one of the following manners:
  • to - allows for defining where the animation is going 'to' from its current position
    e.g. Tweensy.to(item, {x:50, y:100});
  • from - allows for defining where the animation is coming 'from' and will be going to its current position
    e.g. Tweensy.from(item, {x:200, y:150});
  • fromTo - allows for defining where the animation is coming 'from' and going 'to'
    e.g. Tweensy.fromTo(item, {x:200, y:150}, {x:50, y:100});

secondsPerFrameproperty 
secondsPerFrame:Number  [read-write]

Sets how many seconds per frame are added to to each on an ENTER_FRAME when TweensyZero Class's refreshType is of the TweensyZero.FRAME mode.
This property and feature is intended as an alternative to the TweensyZero.TIME (time based animation) mode which can result in jumpy effects. Instead TweensyZero.FRAME ensures that every frame is rendered in the duration of your animation. e.g. If your FLA frame rate is 30 frames per second then set secondsPerFrame to 1 second for every 30 frames (1/30).

public static function from(instance:Object, from:Object, duration:Number = 0.5, ease:Function = null, delayStart:Number = 0, update:Object = null, onComplete:Function = null, onCompleteParams:Array = null):TweensyTimelineZero

Adds a from based tween using the properties defined in the from Object.

Parameters
instance:Object — The instance Object to be tweened or multiple instances if using the type Array e.g. [item1, item2]
 
from:Object — An Object containing the properties you would like to tween from e.g. {x:50, y:25} or this can be relative e.g. {x:'50', y:'-25'} or can be a random position e.g. {x:'-50, 50', y:'-25, 25'}
 
duration:Number (default = 0.5) — The time in secs you would like the tween to run.
 
ease:Function (default = null) — The ease equation you would like to use e.g. Quintic.easeOut.
 
delayStart:Number (default = 0) — The delay you would like to use at the beginning of the tween and every subsequent REPLAY of a tween.
 
update:Object (default = null) — This param is used when tweening a property in a Object which needs to be applied onto another Object each time the tween occurs.
For example to(new DropShadowFilter(), {alpha:0}, myDisplayItem); Will apply the tweening DropShadowFilter onto the DisplayObject 'myDisplayItem'.
 
onComplete:Function (default = null) — The onComplete event handler you would like to fire once the tween is complete.
 
onCompleteParams:Array (default = null) — The params applied to the onComplete handler.

Returns
TweensyTimelineZero — An instance to the TweensyTimelineZero which can used to manage this tween.

stop()method 
public static function stop(instance:* = null, ... props):void

Allows for removing tweens which have been executed by the TweensyZero Class.
Tweens can be stopped via an instance or tween props by the following methods :

  • stop(instance); - stops all property tweens for this instance.
  • stop(instance, "x", "y"); - stops all x,y property tweens for this instance.
  • stop([instance1, instance2]); - stops all property tweens for these instances.
  • stop([instance1, instance2], "x", "y"); - stops all x,y property tweens for these instances.
  • stop(null, "x", "y"); - stops all x,y property tweens.
  • stop(); - stops all tweens, although it is recommeded to use the stopAll method as this is more efficient.
